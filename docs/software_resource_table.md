# Software and Resource Table

Project focus: DNA-seq analysis of red vs black color differences in *Cyprinus carpio* using 10 paired-end WGS samples from BioProject `PRJNA824207`.

Important wording: this is a color-labeled carp dataset, not confirmed ornamental koi. It is still a good biological proxy for a koi/color-difference DNA-seq project because the samples are explicitly labeled red and black.

| Software/resource used | What it does | Biological purpose in this project | Where it came from / how it was installed | Example commands / command pattern used | Data source / input | Output style |
|---|---|---|---|---|---|---|
| NCBI SRA / BioProject `PRJNA824207` | Public archive containing sequencing reads and metadata | Provides confirmed red and black *Cyprinus carpio* WGS samples for comparing color-associated genetic variation | Data source from NCBI SRA; selected runs are listed in `config/sra_runs_red_black_10samples.txt` and metadata in `config/samples_red_black_10samples.tsv` | `RUN_LIST=config/sra_runs_red_black_10samples.txt THREADS=8 MAX_SIZE=20G bash scripts/run_in_env.sh bash scripts/download_sra.sh` | SRA runs `SRR18681855`-`SRR18681859` red and `SRR18681860`-`SRR18681864` black | `.sra` cache files under `data/sra/cache/`; converted FASTQ files under `data/sra/fastq/` |
| Project-local micromamba environment | Runs all software in an isolated virtual environment | Keeps the DNA-seq workflow reproducible and avoids mixing project tools with system tools | Installed locally into `.micromamba/envs/koi-dnaseq`; packages came from Bioconda/Conda-forge using `envs/dnaseq.yml` | `bash scripts/run_in_env.sh fastqc --version`; `bash scripts/run_in_env.sh bash scripts/check_software.sh` | `envs/dnaseq.yml`; local micromamba binary in `.local/bin/micromamba` | Executable environment; version-checked command-line tools |
| SRA Toolkit: `prefetch`, `fasterq-dump` | Downloads SRA accessions and converts `.sra` files to paired FASTQ | Retrieves raw DNA-seq reads for each red and black fish sample | Installed in the `koi-dnaseq` environment from Bioconda | `prefetch --max-size 20G --output-directory data/sra/cache SRR18681855`; `fasterq-dump data/sra/cache/SRR18681855/SRR18681855.sra --split-files --threads 8 --outdir data/sra/fastq --temp /tmp` | NCBI SRA run accessions | Paired FASTQ files such as `SRR18681855_1.fastq.gz` and `SRR18681855_2.fastq.gz` |
| FastQC, raw-read pass | Checks raw sequencing-read quality before trimming | Detects low-quality cycles, adapter contamination, GC bias, overrepresented sequences, and other issues before analysis | Installed in the `koi-dnaseq` environment from Bioconda | `fastqc -t 8 -o results/fastqc/raw data/sra/fastq/SRR18681855_1.fastq.gz data/sra/fastq/SRR18681855_2.fastq.gz` | Raw paired FASTQ files in `data/sra/fastq/` | Per-sample `.html` reports and `.zip` result archives |
| AdapterRemoval | Removes adapter sequence and trims low-quality bases from paired-end reads | Produces cleaner reads so mapping and variant calling are less affected by technical artifacts | Installed in the `koi-dnaseq` environment from Bioconda | `AdapterRemoval --file1 data/sra/fastq/SRR18681855_1.fastq.gz --file2 data/sra/fastq/SRR18681855_2.fastq.gz --basename results/trimmed/SRR18681855 --threads 8 --gzip --trimns --trimqualities --minlength 30` | Raw paired FASTQ files | Trimmed paired reads, singleton reads if any, discarded-read logs, and settings summaries |
| FastQC, post-trim pass | Checks read quality again after AdapterRemoval | Confirms trimming improved quality and reduced adapter contamination before alignment | Installed in the `koi-dnaseq` environment from Bioconda | `fastqc -t 8 -o results/fastqc/trimmed results/trimmed/SRR18681855.pair*.truncated.gz` | Trimmed FASTQ files from AdapterRemoval | Post-trimming `.html` reports and `.zip` archives |
| MultiQC | Aggregates FastQC and other log files into one report | Makes it easier to compare QC across all 10 red/black samples | Installed in the `koi-dnaseq` environment from Bioconda | `multiqc results -o results/multiqc` | FastQC reports, AdapterRemoval logs, alignment/QC summaries | One interactive HTML report plus summary tables |
| *Cyprinus carpio* reference genome FASTA | Provides the coordinate system for read alignment and variant calling | Lets us identify variants relative to a common carp genome and compare red vs black samples at homologous positions | Download from a public genome source such as NCBI RefSeq/GenBank; keep under `references/`; index with Bowtie2, SAMtools, and Picard | `bowtie2-build references/carp.fa references/bowtie2/carp`; `samtools faidx references/carp.fa`; `picard CreateSequenceDictionary R=references/carp.fa O=references/carp.dict` | Reference genome FASTA | Bowtie2 index files, `.fai` FASTA index, and `.dict` sequence dictionary |
| Bowtie2 | Aligns trimmed paired-end reads to the carp reference genome | Places each read on the genome so SNPs/indels can be detected in red and black groups | Installed in the `koi-dnaseq` environment from Bioconda | `bowtie2 -x references/bowtie2/carp -1 results/trimmed/SRR18681855.pair1.truncated.gz -2 results/trimmed/SRR18681855.pair2.truncated.gz -p 8 2> results/alignment/SRR18681855.bowtie2.log \| samtools view -bS - > results/alignment/SRR18681855.unsorted.bam` | Trimmed paired FASTQ files and Bowtie2 reference index | Alignment log and unsorted BAM |
| SAMtools | Converts, sorts, indexes, and summarizes BAM files | Creates coordinate-sorted alignments and basic mapping statistics for each sample | Installed in the `koi-dnaseq` environment from Bioconda | `samtools sort -@ 8 -o results/alignment/SRR18681855.sorted.bam results/alignment/SRR18681855.unsorted.bam`; `samtools index results/alignment/SRR18681855.sorted.bam`; `samtools flagstat results/alignment/SRR18681855.sorted.bam > results/alignment/SRR18681855.flagstat.txt` | BAM files from Bowtie2/SAMtools | Sorted BAM, `.bai` index, text mapping summaries |
| Qualimap | Evaluates mapping quality, coverage, insert sizes, and other BAM-level metrics | Checks whether red and black samples have comparable sequencing and mapping quality before variant comparison | Installed in the `koi-dnaseq` environment from Bioconda | `qualimap bamqc -bam results/alignment/SRR18681855.sorted.bam -outdir results/qualimap/SRR18681855 --java-mem-size=4G` | Sorted BAM files | HTML report, genome coverage plots, text metrics |
| Picard `AddOrReplaceReadGroups` | Adds standardized read-group tags to BAM files | Required by GATK to distinguish samples and correctly process BAMs | Installed in the `koi-dnaseq` environment from Bioconda | `picard AddOrReplaceReadGroups I=results/alignment/SRR18681855.sorted.bam O=results/picard/SRR18681855.rg.bam RGID=SRR18681855 RGLB=SRR18681855 RGPL=DNBSEQ RGPU=SRR18681855 RGSM=SRR18681855` | Coordinate-sorted BAM | Read-group-tagged BAM |
| Picard `MarkDuplicates` | Marks PCR/optical duplicate reads in BAM files | Reduces false-positive variant calls caused by duplicated DNA fragments rather than independent biological evidence | Installed in the `koi-dnaseq` environment from Bioconda | `picard MarkDuplicates I=results/picard/SRR18681855.rg.bam O=results/picard/SRR18681855.dedup.bam M=results/picard/SRR18681855.dup_metrics.txt CREATE_INDEX=true` | Read-group-tagged BAM | Duplicate-marked BAM, BAM index, duplicate metrics text file |
| GATK `HaplotypeCaller` | Calls variants per sample, usually into GVCF mode | Detects SNPs and indels in each fish sample relative to the reference genome | Installed in the `koi-dnaseq` environment from Bioconda | `gatk HaplotypeCaller -R references/carp.fa -I results/picard/SRR18681855.dedup.bam -O results/gatk/gvcf/SRR18681855.g.vcf.gz -ERC GVCF` | Duplicate-marked BAM and reference genome | Per-sample compressed GVCF and index |
| GATK `CombineGVCFs` / `GenotypeGVCFs` | Combines sample GVCFs and jointly genotypes variants | Produces a shared variant set so red and black samples can be compared at the same genomic sites | Installed in the `koi-dnaseq` environment from Bioconda | `gatk CombineGVCFs -R references/carp.fa --variant sample1.g.vcf.gz --variant sample2.g.vcf.gz -O results/gatk/cohort.g.vcf.gz`; `gatk GenotypeGVCFs -R references/carp.fa -V results/gatk/cohort.g.vcf.gz -O results/gatk/cohort.raw.vcf.gz` | Per-sample GVCFs | Joint cohort VCF |
| GATK `VariantFiltration` | Applies hard filters to SNPs/indels | Removes low-confidence variants before comparing allele/genotype patterns between red and black groups | Installed in the `koi-dnaseq` environment from Bioconda | `gatk VariantFiltration -R references/carp.fa -V results/gatk/cohort.raw.vcf.gz -O results/gatk/cohort.filtered.vcf.gz --filter-name QD2 --filter-expression "QD < 2.0" --filter-name FS60 --filter-expression "FS > 60.0" --filter-name MQ40 --filter-expression "MQ < 40.0"` | Raw cohort VCF | Filtered VCF with `FILTER` annotations |
| BCFtools | Indexes, filters, queries, and summarizes VCF files | Creates variant summary tables for SNP counts, genotype patterns, and red-vs-black comparisons | Installed in the `koi-dnaseq` environment from Bioconda | `bcftools index results/gatk/cohort.filtered.vcf.gz`; `bcftools stats results/gatk/cohort.filtered.vcf.gz > results/variants/cohort.bcftools.stats.txt`; `bcftools query -f '%CHROM\t%POS\t%REF\t%ALT[\t%GT]\n' results/gatk/cohort.filtered.vcf.gz > results/variants/genotypes.tsv` | Filtered VCF | VCF indexes, text statistics, genotype tables |
| R / tidyverse / ggplot2 | Performs summary analysis and creates figures | Generates project figures comparing red and black groups, such as QC summaries, variant counts, heterozygosity, SNP density, or PCA-style plots if genotype data are prepared | Installed in the `koi-dnaseq` environment from Conda-forge | `Rscript scripts/plot_variant_summaries.R` | QC tables, BCFtools summaries, genotype tables, sample metadata | `.png`/`.pdf` figures and `.csv` summary tables |
